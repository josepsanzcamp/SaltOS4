1. Panel de control (Dashboard)

1.1. Descripción

El panel de control o dashboard es la pantalla inicial de SaltOS4, donde se pueden mostrar accesos rápidos, separadores, widgets, gráficos y otros elementos visuales. No se trata de una aplicación clásica con formularios o registros, sino de una vista dinámica y personalizable pensada para ofrecer una visión general rápida y acceso directo a funcionalidades frecuentes.

Esta vista centraliza acciones comunes y muestra información relevante de forma resumida.

1.2. Vista principal

El dashboard puede incluir:

  • Botones de acceso rápido a aplicaciones o acciones frecuentes
  • Títulos que agrupan secciones
  • Separadores visuales para organizar la información
  • Widgets que muestran contenido en tiempo real (ej: calendario, correos, gráficos)

1.3. Configuración del dashboard

En la parte superior derecha de la pantalla, hay un botón con un icono de engranaje que abre la aplicación de configuración del dashboard.

Desde allí se pueden personalizar los elementos mostrados:

  • Seleccionar el dashboard activo
  • Añadir o quitar botones
  • Crear títulos y separadores visuales
  • Agrupar elementos en bloques
  • Incluir widgets disponibles (ej: estadísticas, recordatorios, correos)

Esta configuración es específica para cada usuario y se guarda automáticamente.

2. Configuración del dashboard

2.1. Descripción

Esta aplicación permite configurar el contenido que se muestra en el dashboard de SaltOS4. No se trata de una aplicación con registros clásicos, sino de una interfaz visual centrada en el usuario donde se pueden personalizar accesos rápidos, secciones y widgets que aparecen en el panel de control.

Cada usuario puede tener su propio dashboard personalizado, y esta herramienta es su gestor visual.

2.2. Vista de configuración

La pantalla muestra una vista en forma de lienzo donde los elementos del dashboard pueden organizarse de forma dinámica y visual.

Los tipos de elementos que pueden añadirse incluyen:

  • Botones: Enlaces rápidos a aplicaciones o funcionalidades.
  • Separadores: Líneas o espacios para estructurar visualmente el dashboard.
  • Títulos: Textos destacados para agrupar secciones.
  • Widgets: Componentes dinámicos como calendario, correo electrónico, gráficos, estadísticas, etc.

También es posible agrupar elementos, ordenarlos mediante arrastrar y soltar, y reorganizar grupos o columnas.

2.3. Funcionalidades

  • Añadir o eliminar elementos
  • Reordenar mediante arrastrar y soltar
  • Editar títulos y textos de sección
  • Seleccionar qué widgets mostrar
  • Organizar elementos en varias columnas o bloques

Los cambios se guardan automáticamente y afectan solo al dashboard del usuario activo.

2.4. Uso avanzado

Este módulo es ideal para:

  • Crear paneles específicos por perfil (comercial, técnico, administración…)
  • Mostrar solo herramientas y datos consultados con frecuencia
  • Mejorar la eficiencia y claridad visual de la pantalla de inicio

3. Clientes

3.1. Descripción

La aplicación de clientes te permite gestionar tu base de clientes dentro de SaltOS4. Centraliza toda la información relacionada con los clientes: identificación, datos de contacto, código fiscal, dirección y clasificación. También actúa como núcleo para enlazar los clientes con otros módulos como presupuestos, facturas, reuniones y correos.

3.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Nombre: Nombre completo o razón social del cliente.
  • CIF: Código de identificación fiscal del cliente (NIF, CIF, IVA, etc.).
  • Ciudad: Ciudad o localidad del cliente.
  • País: País donde el cliente está registrado o realiza su actividad.
  • Activo: Indica si el cliente está activo en el sistema.

3.3. Vista de formulario

Esta vista se utiliza para crear, editar o consultar una ficha de cliente.

En el modo crear, el formulario está vacío y preparado para introducir nuevos datos.

En el modo visualización, los campos están llenos con la ficha seleccionada y no se pueden editar.

En el modo edición, el formulario está prellenado y permite modificaciones.

El formulario incluye los siguientes campos:

  • Activo: Activa o desactiva la visibilidad del cliente.
  • Nombre: Nombre completo o razón social del cliente.
  • CIF: Código de identificación fiscal del cliente.
  • Dirección: Dirección física principal o de facturación.
  • Ciudad: Ciudad o localidad del cliente.
  • Provincia / Estado: Provincia o estado del cliente.
  • Código postal: Código postal correspondiente a la dirección.
  • País: País de registro.
  • Correo electrónico: Dirección para notificaciones o facturas.
  • Teléfono: Número de contacto principal.
  • Web: Sitio web del cliente.
  • Notas: Anotaciones internas relacionadas con el cliente.
  • Tipo: Categoría a la que pertenece el cliente (ej: habitual, VIP).
  • Archivos: Documentos o imágenes relacionados con el cliente.
  • Notas: Observaciones internas o instrucciones específicas del cliente.

3.4. Eliminación

Los registros pueden eliminarse desde la vista de lista mediante la acción de eliminación. Aparecerá un mensaje de confirmación antes de ejecutar la operación.

Esta acción es irreversible y requiere permisos específicos.

4. Tipos de clientes

4.1. Descripción

Este módulo permite definir distintos tipos de clientes según criterios comerciales, legales u operativos. Clasificar a los clientes ayuda a personalizar la atención, filtrar los datos y aplicar políticas específicas según su categoría.

4.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Nombre: Nombre del tipo de cliente.
  • Descripción: Información adicional sobre el tipo.
  • Activo: Indica si el tipo está disponible para ser asignado.

4.3. Vista de formulario

Esta vista permite gestionar los tipos de cliente.

En el modo crear, se puede agregar un nuevo tipo.

En el modo visualización, los datos se muestran sin posibilidad de edición.

En el modo edición, se pueden modificar los datos existentes.

El formulario incluye los siguientes campos:

  • Activo: Indica si el tipo está activo.
  • Nombre: Nombre que identifica el tipo.
  • Descripción: Explicación o detalle interno del tipo.

4.4. Eliminación

Los tipos de cliente solo pueden eliminarse si no hay ningún cliente asociado a ellos. El sistema pedirá confirmación antes de completar la eliminación.

Esta acción no se puede deshacer.

5. Clientes potenciales

5.1. Descripción

La aplicación de leads se utiliza para registrar y hacer seguimiento de clientes potenciales antes de que se conviertan en clientes reales. Permite recopilar información clave de cada lead, como los datos de contacto, el origen y el estado actual en el proceso de ventas. Este módulo ayuda a organizar la actividad previa a la venta, permitiendo a los equipos comerciales hacer seguimiento efectivo, calificar oportunidades y convertir leads en clientes cuando corresponda.

5.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Título: El título o asunto de la reunión, resumiendo su propósito.
  • Ubicación: Lugar donde se realiza la reunión, o enlace si es virtual.
  • Hora de inicio: Fecha y hora programadas de inicio.
  • Cliente: Cliente asociado a la reunión, si corresponde.

5.3. Vista de formulario

Esta vista se utiliza para crear, editar o consultar un lead.

En el modo crear, el formulario está vacío y listo para introducir nuevos datos.

En el modo visualización, los campos se muestran con la información seleccionada y no se pueden editar.

En el modo edición, el formulario está prellenado y permite realizar modificaciones.

El formulario incluye los siguientes campos:

  • Título: El título o asunto de la reunión, resumiendo su propósito.
  • Ubicación: Lugar donde se realiza la reunión, o enlace si es virtual.
  • Hora de inicio: Fecha y hora programadas de inicio.
  • Hora de finalización: Fecha y hora previstas de finalización.
  • Cliente relacionado: Cliente vinculado a la reunión, si corresponde.
  • Participantes: Lista de usuarios o contactos externos invitados a la reunión.
  • Agenda: Temas o puntos que se tratarán durante la reunión.
  • Temas aprobados: Puntos tratados en la reunión que fueron aprobados.
  • Temas rechazados: Puntos que no fueron aprobados o que se pospusieron.
  • Temas pendientes: Puntos que requieren acciones o decisiones posteriores.

5.4. Eliminación

Los registros pueden eliminarse desde la vista de lista mediante la acción de eliminación. Aparecerá un mensaje de confirmación antes de ejecutar la operación.

Esta acción es irreversible y requiere permisos adecuados.

6. Estados de los leads

6.1. Descripción

Este módulo permite definir los distintos estados por los que puede pasar un cliente potencial dentro del proceso comercial. Los estados ayudan a realizar seguimiento de los leads según su nivel de interés, calificación o decisión final (activo, perdido, convertido, etc.).

6.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Nombre: Nombre del estado.
  • Descripción: Información descriptiva del estado.
  • Activo: Indica si el estado está disponible para su uso.

6.3. Vista de formulario

Esta vista permite gestionar los estados de los leads.

En el modo crear, puedes añadir un nuevo estado.

En el modo visualización, se muestra la información del estado.

En el modo edición, es posible modificar los valores existentes.

El formulario incluye los siguientes campos:

  • Activo: Estado activo o inactivo.
  • Nombre: Título o nombre del estado.
  • Descripción: Explicación sobre el uso o significado del estado.

6.4. Eliminación

Los estados solo pueden eliminarse si no están asignados a ningún lead. El sistema mostrará una confirmación antes de continuar.

Esta acción no se puede deshacer.

7. Reuniones

7.1. Descripción

La aplicación de reuniones permite programar, gestionar y hacer seguimiento de encuentros internos o con clientes dentro de SaltOS4. Incluye información clave como la ubicación, los participantes, la agenda y los temas aprobados, rechazados o pendientes. Este módulo facilita la colaboración y asegura la trazabilidad de las decisiones tomadas durante las reuniones.

7.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Título: Título o asunto de la reunión.
  • Ubicación: Lugar físico o enlace virtual donde se realiza la reunión.
  • Hora de inicio: Fecha y hora programadas de inicio.
  • Cliente: Cliente asociado a la reunión.

7.3. Vista de formulario

Esta vista se utiliza para crear, editar o consultar una reunión.

En el modo crear, el formulario está vacío y listo para registrar una nueva reunión.

En el modo visualización, los campos son de solo lectura y muestran los detalles de la reunión.

En el modo edición, los campos pueden modificarse para actualizar la información de la reunión.

El formulario incluye los siguientes campos:

  • Título: Nombre o asunto de la reunión.
  • Ubicación: Lugar físico o virtual donde se celebrará.
  • Hora de inicio: Fecha y hora de inicio.
  • Hora de finalización: Fecha y hora de finalización.
  • Cliente relacionado: Cliente vinculado a la reunión.
  • Participantes: Usuarios o contactos invitados.
  • Agenda: Lista de puntos a tratar.
  • Temas aprobados: Puntos que han sido aprobados.
  • Temas rechazados: Puntos que no fueron aprobados.
  • Temas pendientes: Puntos que requieren revisión o aprobación futura.

7.4. Eliminación

Las reuniones pueden eliminarse desde la vista de lista mediante la acción correspondiente. Antes de eliminar, el sistema solicitará confirmación.

Esta acción no se puede deshacer y puede requerir permisos específicos.

7.5. Generación de PDF

Desde la vista individual de cada registro (view) se puede generar y descargar un PDF con los datos completos del registro.

Desde la vista de listado (list), se pueden seleccionar múltiples registros mediante checkbox y generar un único documento PDF que los incluye todos.

8. Presupuestos

8.1. Descripción

La aplicación de presupuestos permite generar ofertas comerciales para clientes potenciales o existentes. Incluye campos para definir los datos del cliente, condiciones de pago, líneas de conceptos, impuestos y total. Estos presupuestos pueden convertirse fácilmente en facturas una vez sean aceptados por el cliente.

8.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Cliente: Nombre del cliente destinatario del presupuesto.
  • CIF: Código de identificación fiscal del cliente.
  • Presupuesto: Código o número del presupuesto.
  • Fecha: Fecha de emisión del presupuesto.
  • Total: Importe total con impuestos incluidos.

8.3. Vista de formulario

Esta vista se utiliza para crear, editar o consultar un presupuesto.

En el modo crear, el formulario está vacío y listo para introducir un nuevo presupuesto.

En el modo visualización, los campos son de solo lectura y muestran el detalle del presupuesto.

En el modo edición, se pueden modificar los datos existentes del presupuesto.

El formulario incluye los siguientes campos:

  • Presupuesto: Código del presupuesto.
  • Fecha: Fecha de emisión.
  • Cliente: Nombre o razón social del cliente.
  • CIF: Código fiscal del cliente.
  • Dirección: Dirección de facturación.
  • Ciudad: Localidad.
  • Provincia / Estado: Provincia o región.
  • Código postal: Código postal del cliente.
  • País: País de residencia o registro.
  • Notas: Observaciones internas o condiciones especiales.
  • Conceptos: Líneas detalladas con cantidades e importes.
  • Impuestos: Impuestos aplicables.
  • Totales: Resumen de importe, impuestos y total.
  • Método de pago: Forma de cobro prevista.
  • Válido hasta: Fecha de validez del presupuesto.
  • Estado: Estado actual del presupuesto (pendiente, aceptado, rechazado, etc.).

8.4. Eliminación

Los presupuestos pueden eliminarse desde la vista de lista siempre que no estén vinculados a facturas. El sistema pedirá confirmación antes de completar la acción.

Esta acción no se puede deshacer y puede requerir permisos especiales.

8.5. Generación de PDF

Desde la vista individual de cada registro (view) se puede generar y descargar un PDF con los datos completos del registro.

Desde la vista de listado (list), se pueden seleccionar múltiples registros mediante checkbox y generar un único documento PDF que los incluye todos.

9. Estados de los presupuestos

9.1. Descripción

Este módulo permite definir los diferentes estados por los que puede pasar un presupuesto durante su ciclo de vida. Pueden utilizarse para indicar si un presupuesto está pendiente, aceptado, rechazado o en revisión. Los estados ayudan a hacer seguimiento de los presupuestos y a organizar el proceso comercial.

9.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Nombre: Nombre del estado.
  • Descripción: Explicación o uso del estado.
  • Activo: Indica si el estado está activo y se puede asignar.

9.3. Vista de formulario

Esta vista permite crear, editar o consultar un estado de presupuesto.

En el modo crear, se puede definir un nuevo estado.

En el modo visualización, se muestran los detalles del estado sin posibilidad de edición.

En el modo edición, se pueden modificar los valores existentes.

El formulario incluye los siguientes campos:

  • Activo: Permite activar o desactivar el estado.
  • Nombre: Nombre corto o título del estado.
  • Descripción: Texto descriptivo sobre el uso del estado.

9.4. Eliminación

Los estados pueden eliminarse si no están en uso. Antes de eliminar uno, el sistema pedirá confirmación.

Esta acción no se puede deshacer.

10. Facturas

10.1. Descripción

La aplicación de facturas permite gestionar las facturas emitidas a clientes dentro de SaltOS4. Incluye información del cliente, fecha, estado de pago, líneas de detalle, impuestos y total. También gestiona proformas, vencimientos y registro de pagos.

10.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Cliente: Nombre del cliente facturado.
  • CIF: Código fiscal del cliente.
  • Factura: Código o número de la factura.
  • Fecha: Fecha de emisión.
  • Total: Importe total con impuestos.
  • Cerrada: Indica si la factura ha sido cerrada.
  • Pagada: Indica si la factura ha sido cobrada.

10.3. Vista de formulario

Esta vista se utiliza para crear, consultar o editar una factura.

En el modo crear, el formulario permite generar una nueva factura o proforma.

En el modo visualización, se muestran todos los detalles sin posibilidad de edición.

En el modo edición, se pueden modificar los datos de la factura.

El formulario incluye los siguientes campos:

  • Cerrada: Indica si la factura está finalizada.
  • Pagada: Estado de cobro de la factura.
  • Proforma: Número o referencia de la proforma.
  • Fecha: Fecha de emisión de la factura.
  • Factura: Número final asignado al cerrarse.
  • Cliente: Datos congelados del cliente al momento de emisión.
  • CIF: Código fiscal del cliente.
  • Dirección: Dirección del cliente.
  • Ciudad: Localidad.
  • Provincia / Estado: Región administrativa.
  • Código postal: Código postal.

35.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Usuario: Usuario asociado al token.
  • Creado: Fecha y hora en que se creó el token.
  • Token: Cadena de token utilizada para autenticación o acceso.
  • Activo: Indica si el token es actualmente válido y utilizable.

35.3. Vista de registro

Esta vista muestra toda la información relacionada con un acceso mediante token.

Incluye:

  • Activo: Indica si el token es actualmente válido y utilizable.
  • Usuario: Usuario asociado al token.
  • Creado: Fecha y hora en que se creó el token.
  • Actualizado: Fecha y hora de la última modificación del estado del token.
  • Dirección IP: Dirección desde la cual se utilizó o generó el token.
  • User Agent: Navegador o cliente usado para generar o acceder al token.
  • Token: Cadena usada para autenticación o acceso.
  • Expira: Fecha y hora de expiración del token, tras la cual deja de ser válido.

35.4. Eliminación

Las entradas del registro de tokens no están pensadas para ser eliminadas manualmente.

Forman parte del historial de auditoría y control de accesos del sistema, y pueden eliminarse automáticamente si está configurado.

36. Registro de papelera

36.1. Descripción

La aplicación de registro de papelera almacena todas las eliminaciones realizadas en el sistema, actuando como una papelera de reciclaje para SaltOS4. Permite a administradores y usuarios avanzados inspeccionar qué datos han sido eliminados, cuándo, por quién y desde dónde. Este módulo es clave para la trazabilidad y ayuda a identificar eliminaciones accidentales o acciones no autorizadas.

36.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Usuario: Usuario que realizó la eliminación.
  • Fecha y hora: Momento exacto en que se eliminó el registro.
  • Nombre: Nombre descriptivo del registro o archivo eliminado.

36.3. Vista de registro

Esta vista muestra toda la información registrada sobre un elemento eliminado.

Incluye:

  • ID antiguo: Identificador original del registro antes de su eliminación.
  • Usuario: Usuario que realizó la eliminación.
  • Fecha y hora: Momento exacto en que se eliminó.
  • ID de registro: Identificador interno asignado al registro eliminado.
  • Aplicación: Aplicación o módulo desde donde se eliminó el registro.
  • ID único: Identificador único en todo el sistema del elemento eliminado.
  • Nombre: Nombre descriptivo del registro o archivo eliminado.
  • Tamaño: Tamaño del archivo en bytes, si el elemento era un archivo.
  • Tipo: Tipo MIME o clasificación del elemento eliminado.
  • Archivo: Nombre o ruta original del archivo eliminado.
  • Hash: Suma de verificación utilizada para validar el contenido eliminado.

36.4. Eliminación

Las entradas del registro de papelera no pueden eliminarse directamente desde la interfaz.

SaltOS4 las conserva para fines de responsabilidad y trazabilidad legal, a menos que se configure una depuración automática.

37. Registro de subidas

37.1. Descripción

La aplicación de registro de subidas registra todos los archivos cargados a través de SaltOS4, ya sea de forma manual o automatizada. Ayuda a rastrear el origen, el usuario y el contexto de cada carga, y resulta útil para auditorías, depuración y para verificar que las subidas se procesan correctamente.

37.2. Vista de lista

Los siguientes campos se muestran en la vista de lista:

  • Usuario: Usuario que subió el archivo.
  • Fecha y hora: Momento en que el archivo fue subido al sistema.
  • Aplicación: Módulo o aplicación desde la cual se activó la subida.
  • Nombre: Nombre original del archivo proporcionado por el usuario.

37.3. Vista de registro

Esta vista muestra los detalles completos de una entrada del registro de subidas.

Incluye:

  • Usuario: Usuario que subió el archivo.
  • Fecha y hora: Momento en que el archivo fue subido al sistema.
  • ID único: Identificador único interno asignado a la entrada de subida.
  • Aplicación: Módulo o aplicación desde donde se originó la subida.
  • Nombre: Nombre original del archivo proporcionado por el usuario.
  • Tamaño: Tamaño del archivo subido en bytes.
  • Tipo: Tipo MIME o formato del archivo.
  • Archivo: Nombre del archivo almacenado utilizado internamente por el sistema.
  • Hash: Suma de verificación (ej: MD5 o SHA) para validar la integridad o detectar duplicados.

37.4. Eliminación

Las entradas del registro de subidas normalmente se conservan con fines de trazabilidad histórica.

Si es necesario, se pueden eliminar entradas antiguas manualmente o mediante tareas de mantenimiento automático.
